# Mercur Root Registry

The **Mercur Root Registry** is the authoritative index for the Mercur package ecosystem.

It defines:

- **Authorities** – trusted organizations and maintainers.
- **Namespaces** – ownership and authority assignments.
- **Registry references** – locations of package registries maintained by each authority.

The repository is maintained using human-readable TOML files. These files are automatically validated and compiled into a binary FlatBuffers registry (`registry.mreg`) through GitHub Actions.

## Repository Layout

```text
.
├── authorities/
│   └── *.toml
├── namespaces/
│   └── *.toml
├── registry.mreg
└── .github/
    └── workflows/
```

## registry.mreg

`registry.mreg` is the compiled FlatBuffers representation of the root registry.

Advantages include:

- Fast downloads
- Zero-copy deserialization
- Low memory usage
- High-performance package lookups

Mercur clients download this file directly instead of parsing the individual TOML files.

## Contributing

Contributors should modify the TOML files only.

The compiled `registry.mreg` file is automatically regenerated by the GitHub Actions workflow and committed back to the repository.

Do **not** edit `registry.mreg` manually.
